原来安装过mysql，phpstudy mysql一直没法启动，服务中也没有mysqla。尝试修改端口、查看日志、覆盖复制、赋予权限均无效

解决方案：
1.关闭mysql服务
去服务里关闭即可，如果没有开启则忽略

2.管理员命令下删除mysql服务
mysql为自己的数据库名，默认为mysql
使用此命令后数据库内容不会丢失
1和2顺序可调换

sc delete mysql

3.删除环境变量（之前不成功一直没有删这个）

成功启动

后续尝试将环境变量再加回来仍然可以正常启动

如果需要使用原来的mysql，管理员命令下输入

mysqld --install

再到服务里启动mysql即可
（也可直接输入net start mysql 启动）

后续交替使用时只需要sc delete mysql和mysqld --install切换即可

关于数据库MYSQL服务无法启动的问题可以参考https://blog.csdn.net/weixin_45825865/article/details/122524484?spm=1001.2014.3001.5502
