在 VSCode 中配置 PHP 开发环境

1 XAMPP

XAMPP（Apache+MySQL+PHP+PERL）是一个功能强大的建站集成软件包。这个软件包原来的名字是 LAMPP，但是为了避免误解，最新的几个版本就改名为 XAMPP 了。

官网地址：XAMPP Installers and Downloads for Apache Friends

界面：

2 下载

官网下载地址：Download XAMPP (apachefriends.org)

注意：官网由于网络原因可能下载较慢，我们推荐下面这个网址下载。

其他下载地址：【XAMPP中文版官方下载】XAMPP 8.1.4

3 安装

打开 .exe 文件，直接全程 next，等待安装即可。

注意：模块可按需开启，只用到 PHP，就开启 Apache 即可。

4 配置环境变量

直接搜索系统变量，环境变量=>系统变量=>Path，添加 php.exe 所在目录地址(D:\software\xampp\php)。

注意：系统变量配置完别忘记点击确定。

测试：在 cmd 中输入

php -v

5 Xdebug 下载

要下载与 php 对应的版本，可以在 xmapp 目录下查看 readme 文本文件，如图：

可以看到我们的 php 版本为 8.1.4，而且是 thread safe 版本，对应的是带 TS 的版本。

下载地址：Xdebug: Downloads

把下载的 php_xdebug-3.1.4-8.1-vs16-x86_64.dll 文件，复制到 xampp\php\ext 目录下，然后在 xampp\php 目录的 php.ini 文件末尾添加如下配置：

[xdebug]
zend_extension="D:\software\xampp\php\ext\php_xdebug-3.1.4-8.1-vs16-x86_64.dll"
xdebug.remote_enable = 1
xdebug.remote_autostart = 1

6 在 VSCode 中配置 PHP

首先安装 PHP Debug 插件，如下图：

接着点击 文件 => 首选项 => 设置，输入 php，并点击“在 settings.json 中编辑”，如下图：

然后添加如下配置：

"php.validate.executablePath": "D:/software/xampp/php/php.exe"

最后设置断点调试，如下图

最后将这个 .php 页面放到 xampp\htdocs 目录下，在浏览器输入 http://localhost:80/index.php 即可访问这个页面，如下图：

访问的端口号看这里：

注意：这里我们在浏览器中输入的是 http://localhost:80/index.php，然而在浏览器地址中为什么显示的却是 localhost/index.php 呢？这是因为浏览网页服务默认的端口号都是 80，因此只需输入网址即可，不用输入": 80"了。另外如果 .php 页面为 index 命名，则只需要在浏览器输入 localhost 即可，因为服务器会自动为我们找到 index.php 这个页面。

7 补充

推荐插件 PHP Server

这个插件可以让 .php 文件直接右击运行于浏览器中，不需要借助于 xampp 的 Apache。

注意：该插件用于个人测试及小网站的开发是没有任何问题的，不过大型应用还是要配合 Nginx 或 Apache 以达到最高效率。
