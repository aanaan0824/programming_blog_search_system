SQL模糊搜索的4种匹配模式：

1. % :用来替代任意长度和任意类型的字符串（%中文%）；

eg: select * from table where keyname like

%白%：检索字符串中含有‘白’的；

%白% and %痴%：检索含‘白’和‘痴’的，不分前后；

%白%吃%：白在吃前

code eg:(" select * from s_xml where course_id like \"%%1%\””).arg("白");

2. _ :用来代替任意的单个字符；

eg:'白_'

3. []: 取[]内的任意一个字符与后面组合；

相当于[{X,Y,Z...}]与后面部分的直积，“[什么啊]月” ={“什月”，“么月”，“啊月”}；

连续的数字和字母可以缩写，[1-4] =[1234];

4. [^]: 与3类似，取[]内部分的补集；

需要转义的字符用[]括起来，如‘[’ =>[[];

'_月' =》检索以月结尾的；

'月_'=》 检索以月开头的；

‘月’=》检索含月的；
