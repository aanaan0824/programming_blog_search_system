有关问题解决

1. 由于找不到msvcr110 dll无法继续执行


解决办法
运行的php5.6为64位，需要安装64位Visual C++ Redistributable for Visual Studio 2012。

下载网址：
< https://www.microsoft.com/zh-CN/download/details.aspx?id=30679 >



2. 切换中文后，出现错误


解决办法
切换语言，实际只是修改了配置文件。所以，只要把配置文件还原就没有问题了。
配置文件有两个， wampmanager.conf 和 wampmanager.ini，都在应用根目录下。
wampmanager.conf 文件，第二行内容就是设置语言，先还原成english

  • 更改前：
[main]
language = "chinese"
wampserverVersion ="3.2.6"
  • 更改后：
[main]
language = "english"
wampserverVersion ="3.2.6"
...

wampmanager.ini中注释掉prompt相关配置。在行首添加分号;

再次启动，会提示出错：

不用管它，直接确定。然后应用会自动更新配置文件。
自此完美解决。

Wampserver如何添加mongodb扩展

输入http://localhost/?phpinfo=-1：

如果是以上图片，此时PHP的版本是7.4.26，为了学习方便，在这里我更改了我的PHP版本为5.6.40。

再输入http://localhost/?phpinfo=-1，页面改变了。

1.下载php_mongodb.dll扩展

php官网上面mongodb的版本有很多,需要下载适合环境的package,下载链接如下:
https://pecl.php.net/package/mongodb/1.5.0/windows
我选了1.5.0稳定版,接着选择跟PHP拓展环境相匹配的DLL包:5.6 Thread Safe (TS) x64

修改:
以上是php7以后才使用的mongodb扩展，现更改一下，对于php7以前的版本使用mongodb的扩展下载链接：
https://pecl.php.net/package/mongo/1.6.14/windows

2.安装mongodb扩展

  • 下载完成后将压缩包里面的php_mongodb.dll解压到PHP扩展目录,比如：D:\CodeApp\Wampserver3.2.0\bin\php\php5.6.40\ext
  • 接着打开D:\CodeApp\Wampserver3.2.0\bin\php\php5.6.40\phpForApache.ini,在文件末尾添加一行：extension=php_mongodb.dll保存后重启apache服务。

修改:
D:\CodeApp\Wampserver3.2.0\bin\php\php5.6.40\phpForApache.ini,在文件末尾添加一行：extension=php_mongo.dll保存后重启apache服务。

3.修改系统变量

因为mongodb需要依赖libsasl.dll扩展指向，扩展无法找到libsasl.dll的位置就会导致mnogodb扩展无法使用，于是需要在系统环境变量中增加libsasl.dll的路径：D:\CodeApp\Wampserver3.2.0\bin\php\php5.6.40

4.验证mongodb扩展是否安装成功

保存好环境变量之后关掉wamp再打开，在浏览器中输入localhost，点击进入phpinfo()，如果能在页面上能看到mongodb模块即为安装成功。
